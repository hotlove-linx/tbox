## 1.开发要求
使用agent team模式创建一个团队，团队成员包括：
- backend后端和admin-backend后端开发
- 前端开发
- 桌面端开发
- 测试
**角色要求：**
1. backend后端负责开发backend后端服务
2. admin-backend后端开发负责运营管理后台的后端开发
3. 前端开发负责管理后台的前端界面开发
4. 桌面端开发负责桌面端软件的开发
5. 测试负责测试所有后端和前端和桌面端代码

**开发注意事项：**
1. backend后端开发需要结合`需求文档-细化版.md`和prototype目录下`index.html`原型文件进行开发
2. admin-backend后端开发需要结合`后台运营管理平台-需求文档.md`和prototype目录下`admin.html`原型文件进行开发
3. 桌面端开发需要结合`需求文档-细化版.md`和prototype目录下`index.html`原型文件进行开发
4. 前端开发需要结合`后台运营管理平台-需求文档.md`和prototype目录下`admin.html`原型文件进行开发

## 2.技术要求
### 2.1 后端
- backend后端和admin-backend后端均使用golang开发
- 数据库使用mysql，web框架使用gin，数据库框架使用gorm
- 缓存使用redis
- 用户登录数据保存在redis中
- 服务端口号，日志等级，数据库连接信息等配置信息保存在配置文件中，配置文件使用yaml格式
### 2.2 前端
- 桌面端开发：vue3+ts+vite+electron
- 管理后台前端：vue3+ts+vite

## 3.项目工程要求
- 服务后端代码放在backend目录下
- 运营后端代码放在admin-backend目录下
- 桌面端代码放在desktop目录下
- 管理后台前端代码放在admin-frontend目录下
