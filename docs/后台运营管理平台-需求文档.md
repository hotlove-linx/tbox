# TBox 后台运营管理平台 - 需求文档

> 版本：v1.0
> 日期：2026-03-18
> 状态：需求设计阶段
> 关联文档：TBox 桌面端软件需求文档 v1.1

---

## 1. 项目概述

### 1.1 项目背景

TBox 桌面端软件商城需要一个配套的**后台运营管理平台**（Web 端），供运营团队和管理员对平台内容、用户、审核、数据等进行统一管理。该平台是 TBox 生态的核心支撑系统，确保商城内容质量、用户体验和运营效率。

### 1.2 目标用户

| 角色 | 说明 |
|------|------|
| 超级管理员 | 拥有所有权限，可管理系统配置和管理员账户 |
| 运营管理员 | 负责首页推荐、Banner 管理、分类管理等运营工作 |
| 审核员 | 负责软件上架审核、评论审核等内容审核工作 |
| 客服人员 | 处理用户反馈、投诉和问题工单 |

### 1.3 技术定位

- Web 端管理后台，基于浏览器访问
- 响应式布局，最低支持 1280x720 分辨率
- 与 TBox 桌面端共用同一套后端 API 和数据库

## 2. 整体布局与导航结构

### 2.1 布局说明

采用经典的后台管理系统布局：**顶部导航栏 + 左侧菜单栏 + 右侧内容区**

- **顶部导航栏**：平台 Logo（TBox Admin）、面包屑导航、全局搜索、消息通知（角标）、管理员信息（头像 + 姓名 + 角色 + 下拉菜单）
- **左侧菜单栏**：支持折叠/展开，分组菜单导航，菜单项带图标
- **内容区**：页面标题 + 操作区 + 主体内容，支持 Tab 多页签模式

### 2.2 导航菜单结构

## 3. 功能模块详细设计

### 3.1 仪表盘

#### 3.1.1 核心指标卡片

页面顶部展示核心运营指标，每张卡片包含指标名称、当前数值、环比变化（百分比 + 上升/下降箭头）：

| 指标卡片 | 内容 |
|----------|------|
| 总用户数 | 注册用户总数 + 今日新增 |
| 总软件数 | 已上架软件总数 + 待审核数量 |
| 今日下载量 | 今日下载总次数 + 较昨日变化 |
| 活跃用户 | 今日活跃用户数（DAU）+ 较昨日变化 |

#### 3.1.2 数据图表区

- **下载量趋势图**：折线图，展示近 7 天 / 30 天的每日下载量，支持切换时间范围
- **用户增长趋势图**：折线图，展示近 7 天 / 30 天的新增注册用户数
- **软件分类分布**：饼图 / 环形图，展示各分类下的软件数量占比
- **热门软件 Top 10**：横向柱状图，展示下载量前 10 的软件

#### 3.1.3 待办事项区

- 待审核软件数量（可点击跳转至审核列表）
- 待处理举报数量
- 待回复用户反馈数量

### 3.2 软件管理模块

#### 3.2.1 软件列表

**页面工具栏：**
- 搜索框：按软件名称 / 开发者搜索
- 筛选条件：分类（下拉）、审核状态（下拉）、可见性（下拉）、上架时间范围（日期选择器）
- 操作按钮：导出列表（Excel）、批量操作下拉

**表格列定义：**

| 列名 | 内容 |
|------|------|
| ID | 软件唯一标识 |
| 软件信息 | 图标 + 名称 + 版本号 |
| 分类 | 所属分类标签 |
| 开发者/上传者 | 开发者名称（点击可查看上传者信息） |
| 可见性 | 公开（蓝色）/ 私有（灰色）/ 链接分享（橙色） |
| 审核状态 | 已通过（绿色）/ 审核中（黄色）/ 已拒绝（红色）/ 待审核（灰色） |
| 下载量 | 累计下载次数 |
| 评分 | 平均评分（星级 + 数值） |
| 文件大小 | 安装包大小 |
| 上架时间 | 审核通过并上架的时间 |
| 操作 | 查看详情 / 编辑 / 上架 / 下架 / 删除 |

**支持功能：**
- 分页展示，可配置每页条数（10/20/50/100）
- 点击列头排序（下载量、评分、上架时间）
- 批量选择：批量上架 / 批量下架 / 批量删除

#### 3.2.2 软件详情查看/编辑

管理员可查看和编辑软件的完整信息：

- **基本信息**：软件名称、版本号、分类、开发者、简介、详细描述
- **文件信息**：安装包文件名、文件大小、下载地址、SHA-256 校验值
- **展示资源**：软件图标（支持替换）、软件截图（支持增删排序）
- **运营属性**：可见性（公开/私有/链接分享）、是否推荐、推荐排序权重、标签管理
- **审核信息**：审核状态、审核记录（审核人、时间、结果、备注）
- **版本历史**：所有版本列表，每个版本的更新日志、文件信息
- **统计数据**：下载量趋势、评分分布、用户评价列表

#### 3.2.3 软件上下架

- **上架**：审核通过的软件可手动上架至商城，上架后客户端可见
- **下架**：已上架软件可下架，下架后客户端不可搜索和浏览，但已下载用户不受影响
- **下架需填写原因**（如违规、开发者申请、质量问题等）
- 上架/下架操作记录在操作日志中

#### 3.2.4 分类管理

**表格列定义：**

| 列名 | 内容 |
|------|------|
| ID | 分类唯一标识 |
| 分类名称 | 分类显示名称 |
| 分类图标 | 分类对应图标 |
| 排序权重 | 控制前端展示顺序（数值越大越靠前） |
| 软件数量 | 该分类下的软件总数 |
| 状态 | 启用 / 禁用 |
| 创建时间 | 分类创建时间 |
| 操作 | 编辑 / 启用/禁用 / 删除 |

**操作规则：**
- 新增分类：填写名称、图标、排序权重
- 编辑分类：修改名称、图标、排序权重
- 禁用分类：禁用后前端不展示该分类标签，但已有软件不受影响
- 删除分类：分类下有软件时不可删除，需先迁移软件至其他分类

#### 3.2.5 标签管理

管理软件特性标签（如"免费"、"开源"、"精品"、"新上架"、"编辑推荐"等）：

| 列名 | 内容 |
|------|------|
| ID | 标签唯一标识 |
| 标签名称 | 标签显示文字 |
| 标签颜色 | 标签展示颜色 |
| 标签类型 | 系统标签（自动打标）/ 运营标签（手动打标） |
| 关联软件数 | 使用该标签的软件数量 |
| 状态 | 启用 / 禁用 |
| 操作 | 编辑 / 禁用 / 删除 |

### 3.3 内容运营模块

#### 3.3.1 Banner 管理

管理首页轮播 Banner 位（3~5 个位置）：

**表格列定义：**

| 列名 | 内容 |
|------|------|
| 排序 | 拖拽排序手柄 + 序号 |
| Banner 预览 | 缩略图预览 |
| 标题 | Banner 标题文字 |
| 关联软件 | 关联的软件名称（点击跳转软件详情） |
| 展示时间 | 开始时间 ~ 结束时间 |
| 状态 | 展示中（绿色）/ 已过期（灰色）/ 未开始（蓝色）/ 已禁用（红色） |
| 操作 | 编辑 / 启用/禁用 / 删除 / 预览 |

**新增/编辑 Banner：**
- Banner 图片上传（推荐尺寸，支持裁剪预览）
- 标题、副标题文字
- 关联软件选择（搜索选择已上架软件）
- 跳转类型：软件详情页 / 分类页 / 外部链接
- 展示时间段（开始 ~ 结束日期时间）
- 排序权重

#### 3.3.2 推荐管理

管理首页"热门推荐"和"下载排行榜"两个区域的内容：

**热门推荐：**
- 推荐方式：手动推荐 + 算法推荐
- 手动推荐：从已上架软件中搜索选择，设置推荐位置和排序权重
- 算法推荐：基于下载量、评分、更新时间的加权排序（可配置权重）
- 推荐列表管理：添加推荐 / 移除推荐 / 调整排序 / 设置展示有效期

**下载排行榜：**
- 排行周期配置：实时 / 日榜 / 周榜 / 月榜
- 排行规则：按下载量降序，支持配置是否排除某些软件
- 手动干预：支持置顶 / 移除特定软件
- 前端默认展示数量配置

#### 3.3.3 公告管理

**表格列定义：**

| 列名 | 内容 |
|------|------|
| ID | 公告唯一标识 |
| 标题 | 公告标题 |
| 类型 | 系统公告 / 活动通知 / 版本更新 / 维护通知 |
| 发布状态 | 草稿 / 已发布 / 已撤回 |
| 推送方式 | 应用内通知 / 弹窗通知 |
| 目标用户 | 全部用户 / 指定用户群 |
| 发布时间 | 发布或定时发布时间 |
| 操作 | 编辑 / 发布 / 撤回 / 删除 / 预览 |

**新增/编辑公告：**
- 标题、内容（富文本编辑器）
- 公告类型选择
- 推送方式配置
- 定时发布设置
- 目标用户范围

### 3.4 审核中心模块

#### 3.4.1 软件审核

**页面工具栏：**
- 状态 Tab 切换：待审核 / 审核中 / 已通过 / 已拒绝（每个 Tab 显示数量角标）
- 搜索框：按软件名称搜索
- 筛选：分类、提交时间范围

**待审核列表：**

| 列名 | 内容 |
|------|------|
| ID | 审核单唯一标识 |
| 软件信息 | 图标 + 名称 + 版本号 |
| 提交者 | 上传用户昵称（点击查看用户信息） |
| 分类 | 软件所属分类 |
| 文件大小 | 安装包大小 |
| 安全扫描 | 扫描状态（通过/风险/未扫描） + 扫描报告链接 |
| 提交时间 | 提交审核的时间 |
| 操作 | 审核（进入审核详情） |

**审核详情页：**
- 软件完整信息查看（名称、描述、截图、文件信息等）
- 安全扫描报告查看（病毒扫描结果、恶意代码检测结果）
- 文件下载测试（管理员可下载安装包进行人工测试）
- 历史审核记录（如有多次提交）
- 审核操作区：
  - **通过**：确认上架，可填写备注
  - **拒绝**：必须填写拒绝原因（用户可在客户端查看原因并修改重提）
  - **转交**：转交给其他审核员处理
  - **标记风险**：通过但标记风险，后续重点关注

**审核规则配置：**
- 自动安全扫描：上传后自动触发病毒扫描和恶意代码检测
- 扫描不通过自动拒绝
- 审核时限：超时未处理自动提醒

#### 3.4.2 评论审核

管理用户对软件的评价内容：

**表格列定义：**

| 列名 | 内容 |
|------|------|
| ID | 评论唯一标识 |
| 评论内容 | 评论文字（截断展示，hover 查看全部） |
| 评分 | 星级评分 |
| 评论用户 | 用户昵称 |
| 关联软件 | 软件名称 |
| 发布时间 | 评论发布时间 |
| 状态 | 正常 / 已屏蔽 / 待审核 |
| 操作 | 查看详情 / 屏蔽 / 恢复 / 删除 |

**审核机制：**
- 敏感词自动过滤（命中敏感词的评论自动进入待审核状态）
- 人工审核：查看评论内容，决定放行或屏蔽
- 屏蔽后用户侧不可见，但数据保留

#### 3.4.3 举报处理

处理用户对软件或评论的举报：

| 列名 | 内容 |
|------|------|
| ID | 举报单唯一标识 |
| 举报类型 | 软件举报 / 评论举报 |
| 举报对象 | 被举报的软件名称或评论内容 |
| 举报原因 | 恶意软件 / 侵权 / 虚假信息 / 色情暴力 / 其他 |
| 举报描述 | 举报者补充说明 |
| 举报人 | 举报用户昵称 |
| 举报时间 | 举报提交时间 |
| 处理状态 | 待处理 / 处理中 / 已处理 |
| 处理结果 | 有效（已处理）/ 无效（驳回） |
| 操作 | 查看详情 / 处理 |

**处理操作：**
- 查看被举报内容详情
- 标记处理结果：有效（执行下架/屏蔽/警告等措施）或无效（驳回举报）
- 填写处理说明

### 3.5 用户管理模块

#### 3.5.1 用户列表

**页面工具栏：**
- 搜索框：按邮箱 / 昵称搜索
- 筛选条件：注册时间范围、账户状态（正常/禁用/注销）
- 导出按钮

**表格列定义：**

| 列名 | 内容 |
|------|------|
| ID | 用户唯一标识 |
| 用户信息 | 头像 + 昵称 + 邮箱 |
| 注册时间 | 账户注册日期 |
| 上传软件数 | 该用户上传的软件总数 |
| 总下载量 | 该用户软件的累计被下载次数 |
| 空间使用 | 已用容量 / 总容量 |
| 账户状态 | 正常（绿色）/ 已禁用（红色）/ 已注销（灰色） |
| 最后登录 | 最后一次登录时间 |
| 操作 | 查看详情 / 禁用/启用 / 调整空间容量 |

#### 3.5.2 用户详情

点击用户进入详情页，包含：

- **基本信息**：头像、昵称、邮箱、注册时间、最后登录时间、账户状态
- **空间信息**：总容量、已用容量、容量调整（管理员可手动调整用户空间上限）
- **上传软件列表**：该用户上传的所有软件（含各状态）
- **评论记录**：该用户发表的所有评论
- **操作记录**：登录记录、上传记录、下载记录
- **管理操作**：禁用账户（需填写原因）、恢复账户、调整空间容量、重置密码

#### 3.5.3 用户反馈

处理用户通过客户端"反馈与帮助"提交的反馈：

| 列名 | 内容 |
|------|------|
| ID | 工单唯一标识 |
| 反馈类型 | Bug 反馈 / 功能建议 / 投诉 / 其他 |
| 标题 | 反馈标题 |
| 内容摘要 | 反馈内容（截断展示） |
| 提交用户 | 用户昵称 + 邮箱 |
| 提交时间 | 反馈提交时间 |
| 处理状态 | 待处理 / 处理中 / 已回复 / 已关闭 |
| 处理人 | 分配的客服人员 |
| 操作 | 查看 / 回复 / 分配 / 关闭 |

**处理流程：**
- 查看反馈详情（含用户附带的截图）
- 分配给对应客服/技术人员
- 回复用户（用户可在客户端收到回复通知）
- 标记关闭

### 3.6 数据统计模块

#### 3.6.1 下载统计

- **下载量总览**：今日下载量、本周下载量、本月下载量、累计下载量
- **下载趋势图**：折线图，支持按日/周/月维度查看，支持自定义时间范围
- **分类下载分布**：柱状图/饼图，各分类的下载量占比
- **时段分布**：柱状图，展示一天内各时段的下载量分布

#### 3.6.2 用户统计

- **用户总览**：总注册用户数、今日新增、本周新增、本月新增
- **用户增长趋势**：折线图，支持按日/周/月维度查看
- **用户活跃度**：DAU（日活）、WAU（周活）、MAU（月活）趋势图
- **用户留存率**：次日留存、7 日留存、30 日留存

#### 3.6.3 软件排行

- **下载排行**：按下载量降序，支持按时间范围筛选（全部/本周/本月/本年）
- **评分排行**：按平均评分降序，筛选最低评价数量门槛
- **新增排行**：按上架时间排序，展示最新上架的软件
- **增长排行**：按下载量增速排序（本周/本月环比增长最快的软件）

所有排行支持导出 Excel。

## 4. 管理员账户体系

### 4.1 管理员登录

- 独立登录页面，与用户端登录系统隔离
- 邮箱 + 密码登录
- 支持验证码（图形验证码 / 邮箱验证码）
- 登录失败 5 次锁定账户 30 分钟
- 登录有效期 8 小时，超时需重新登录
- 支持"记住我"延长至 7 天

### 4.2 安全机制

- 管理员密码要求：最少 10 位，包含大小写字母 + 数字 + 特殊字符
- 首次登录强制修改初始密码
- 密码有效期 90 天，过期强制更换
- 关键操作二次确认（删除软件、禁用用户、修改系统配置等）
- 支持 IP 白名单配置（可选）
- 所有操作记录审计日志

## 8. 非功能性需求

### 8.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | ≤ 2 秒 |
| 列表查询响应 | ≤ 1 秒 |
| 图表数据加载 | ≤ 3 秒 |
| 导出操作 | 10000 条数据导出 ≤ 10 秒 |

### 8.2 安全要求

- 管理员登录与用户登录系统完全隔离
- 所有 API 接口进行权限校验
- 通信全程 HTTPS
- 关键操作二次确认
- 操作审计日志完整记录
- 支持 IP 白名单限制后台访问（可选）
- 防止 XSS、CSRF、SQL 注入等常见攻击

### 8.3 兼容性要求

- 浏览器：Chrome 90+、Edge 90+、Firefox 90+
- 分辨率：最低支持 1280 x 720

### 8.4 可用性要求

- 响应式布局，适配不同屏幕宽度
- 表格支持列宽调整、固定列
- 表单校验实时反馈
- 操作结果有明确提示（成功/失败 Toast）
- 删除等危险操作有确认弹窗
- 支持快捷键操作
